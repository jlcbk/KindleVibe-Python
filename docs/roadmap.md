# 路线图

这份路线图记录 InkDash（墨板）后续可以继续迭代的方向。它不是承诺清单，优先级应根据真实使用反馈调整。

## 近期方向

### 看板交互

- 在设置页或主看板上提供 preset 快捷入口。
- 在主看板上显示当前状态距离上次更新经过了多久。
- 为 `/status.txt` 和 `/presets.txt` 增加更紧凑的短版输出选项。
- 增加一个只显示协作状态、不显示 Codex 用量的极简页面。

### CLI 和自动化

- 增加批量事件导入能力，方便脚本一次写入多条关键事件。
- 为常见 agent 工作流补充更多 preset，例如 `testing`、`deploying`、`paused`。
- 给 `make preset-*` 增加更明确的失败提示，区分服务不可达和 token 错误。

### 文档和示例

- 补充真实 Kindle 型号上的兼容性记录。
- 补充反向代理部署示例，但仍然明确公网暴露的安全边界。
- 增加常见 CI 或本地脚本的接入示例。

## 中期方向

### 多项目状态

当前 `inkdash_status.json` 只保存一份状态。后续可以考虑：

- 按项目保存多个状态。
- 主页面显示当前项目，API 支持选择项目。
- 保持 `/status.txt` 仍能输出默认项目的紧凑摘要。

### 状态历史

当前事件历史只保留在状态对象中。后续可以考虑：

- 将事件追加到独立历史文件。
- 提供最近 N 条历史的只读 API。
- 支持按天轮转历史，避免运行时间长后文件过大。

### 更强的健康检查

- 区分服务自身健康、Vibe 心跳健康、Codex 数据健康。
- 增加 `degraded` 状态，而不是永远返回 `ok`。
- 提供纯文本健康端点，方便旧设备和 shell 脚本读取。

## 长期方向

### 轻量集成

- 与常见 agent runner 的日志或状态文件做只读集成。
- 支持从 GitHub PR 状态、CI 状态或本地任务文件生成 Vibe 状态。
- 保持集成可选，避免破坏标准库优先和简单部署的核心边界。

### 更好的 Kindle 体验

- 为不同 Kindle 分辨率提供布局档位。
- 增加夜间/白天高对比主题。
- 提供截图或手动验证清单，记录各型号浏览器差异。

## 当前不优先做的事情

- 不优先引入数据库；当前本地 JSON 文件足够简单。
- 不优先引入前端构建系统；旧 Kindle 兼容性和部署简单性更重要。
- 不优先承诺公网安全部署；默认场景仍是可信局域网。
- 不优先把 Codex 用量监控做成通用监控系统；核心目标是电子墨水屏上的 Codex 用量和协作状态展示。
